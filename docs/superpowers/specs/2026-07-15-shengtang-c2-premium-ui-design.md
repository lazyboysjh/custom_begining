# 圣堂初遇 C2 高级玻璃界面设计

## 目标

封面与状态栏统一采用“现代玻璃空间感 + 大魏式实体按钮结构”。升级重点不是替换颜色，而是建立完整材质层级、控件状态、边缘反射、暗部深度和跨端布局规则，使界面在酒馆消息楼层内呈现为成熟产品。

本次只调整视觉、布局、动效与用户文案，不改变已经验证的开局事务、生成流程、MVU 多角色结构和变量更新规则。

## 视觉系统

- 主背景使用中性冷黑，容器通过半透明表面、真实模糊、轻微饱和度和局部暖金反射形成空间感。
- 主容器使用细外边框、顶部内高光、深层环境阴影；内部面板降低阴影强度，避免所有区域都像浮卡。
- 香槟金只用于主要操作、当前选择、进度和关键状态，不铺满整个界面。
- 封面与状态栏共享颜色、边框、圆角、阴影、缓动和控件尺寸 token。
- 主要容器圆角为 8px，紧凑控件为 6px 至 8px；不使用夸张胶囊按钮。

## 按钮与控件

所有按钮必须同时具备以下材质层：

1. 可见但克制的实体边框。
2. 从上到下的玻璃金属明暗变化。
3. 顶部一像素反射高光。
4. 底部暗边和外部投影，表达控件厚度。
5. 悬停时上浮、边缘增亮和阴影加深。
6. 按压时下沉、轻微缩放并切换为内阴影。
7. 清晰的默认、悬停、按压、选中、禁用、加载和键盘焦点状态。

主要按钮采用香槟金属玻璃表面，文字保持深色高对比。次按钮采用冷灰玻璃表面。图标按钮使用稳定正方形尺寸并提供 `title` 或可访问名称。禁用态降低对比度且取消位移反馈，但文字仍可辨认。

输入框、标签页、角色卡片、模式卡片和角色切换器沿用相同边缘结构，不允许出现按钮很精致而其他控件仍是单层底色的割裂感。

## 封面

- 首屏、相遇方式、主角设定、角色选择和生成中页面使用同一玻璃外壳与操作区。
- 首屏保持品牌名“圣堂初遇”为第一视觉信号，并露出进入流程的主按钮。
- 相遇方式和角色卡片使用玻璃面板、细高光和明确选中态；不再使用粗描边或单纯颜色填充表示选中。
- 表单控件使用统一焦点环、表面材质和稳定高度。
- 底部操作区在桌面横向排列，在手机端平均分配可用宽度；错误文案不挤压或覆盖按钮。
- 生成中页面只显示生成状态、必要进度和取消操作，不暴露接口、变量或事务细节。

## 状态栏

- 顶部只展示当前场景、回合以及必要的工具按钮。
- 多角色入口来自 `世界.出场角色`。桌面和平板横向展示角色切换器，手机保持单行横向滚动或紧凑切换，不压缩角色名到不可辨认。
- 当前角色面板展示姓名、作品、阶段、关系、核心数值、当前心态与目标；低价值字段不重复展示。
- 数值块使用内凹玻璃槽和细进度线，不使用高噪声发光。
- 行动按钮与封面使用完全相同的按钮组件状态。
- 空数据、加载失败和无角色场景使用简短可行动文案，不能显示变量路径或开发说明。

## 用户文案

正式界面允许显示：

- 品牌、场景、回合、角色身份和关系信息。
- 玩家当前需要作出的选择。
- 简短的加载、失败、重试和取消提示。
- 对当前状态有直接意义的心态、目标和边界。

正式界面禁止显示：

- CDN、commit、版本号、构建标识和调试状态。
- MVU、API、JSON Patch、变量路径、楼层号或初始化成功提示。
- “正在初始化变量并请求模型”等内部过程说明。
- 面向开发者的教程、功能描述、设计说明和测试文案。
- 无法指导用户行动的英文眉题、装饰性编号或不明所以的缩写。

技术错误写入控制台；用户界面只显示“状态暂不可用”“生成失败，请重试”等可理解文案。

## 响应式布局

### 桌面

- 封面保持 16:9 外壳，流程页使用信息区与操作区双栏，充分利用横向空间。
- 状态栏使用紧凑双栏或横向指标布局，最大宽度受控，不出现大面积空区。

### 平板

- 在容器宽度不足时从双栏平滑切换为单列，不依赖设备名称或 viewport 字号缩放。
- 角色切换器允许横向滚动，工具按钮保持固定尺寸。

### 手机

- 封面使用 9:16 外壳；卡片和表单转为单列，主次操作按钮共享可用宽度。
- 状态栏单列排列，数值块根据最小宽度在两列与一列间切换。
- 触控目标不小于 40px，正文和按钮文字不截断关键含义。

### 通用约束

- 不使用 `vh` 强撑酒馆消息楼层。
- 页面不得横向溢出，不使用会造成双层滚动的固定高度列表。
- 动态文本允许换行或省略，但不能覆盖相邻控件。
- 支持安全区和 `prefers-reduced-motion`。
- 断点由容器可用宽度决定，必须覆盖 360px 手机、768px 平板和 1280px 桌面审查。

## 动效

- 按钮与紧凑控件反馈为 180ms 至 240ms。
- 页面和面板进入为 320ms 至 480ms，使用减速缓动。
- 角色切换采用短距离淡入和轻微位移，不整页闪烁。
- 加载动效保持低亮度，不使用持续高强度发光。
- 减少动态模式下关闭非必要位移、循环和模糊变化。

## 验收与发布

- 回归测试必须保持开局事务、MVU 多角色、消息楼层读取和 CDN 固定逻辑通过。
- 静态审查增加按钮完整状态、禁止文案、共享视觉 token 和无横向溢出检查。
- 浏览器分别审查封面所有步骤页和状态栏，覆盖桌面、平板、手机。
- 浏览器检查按钮悬停、按压、禁用、加载、选中、键盘焦点以及多角色切换。
- 检查控制台错误、文本重叠、裁切、嵌套滚动和 reduced-motion。
- 发布流程仍为：静态 UI commit 并推送，角色卡 CDN 固定该 commit，重建卡片后再次提交并推送，最后核验固定 CDN 返回 200。
