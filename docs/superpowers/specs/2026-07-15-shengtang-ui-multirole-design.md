# 圣堂初遇 UI 与多角色 MVU 重构设计

## 目标

重做封面与状态栏，使其具备统一的黑金磨砂产品视觉、成熟的多端布局和克制的用户文案。同时把当前只能表达单个“初遇”对象的 MVU 结构升级为可支持同一场景多角色同时出现、分别更新和独立展示的数据模型。

## 数据结构

采用“当前场景有序数组 + 永久角色对象表”：

```yaml
世界:
  场景: 圣言堂
  出场角色:
    - 牧濑红莉栖

角色:
  牧濑红莉栖:
    作品: 命运石之门
    是否自定义: false
    污秽类型: 异界残留
    污秽度: 40
    信任: 15
    好感度: 10
    堕落值: 5
    依存度: 0
    仪式阶段: 初见
    与user关系: 陌路
    当前心态: 保持戒备并判断环境
    当前目标: 确认污秽来源并保全自身
    对user判断: 能力与动机待验证
    当前边界: 不接受未经说明的私密接触
```

- `世界.出场角色` 是当前场景的有序姓名数组，只决定状态栏显示与切换范围。
- `角色` 是以姓名为键的永久对象表，角色离场时档案仍保留。
- Schema 对出场数组执行去空、去重，并过滤不存在于角色表的姓名。
- 角色对象中的五项数值统一限制为 `0..100`。
- 旧结构 `初遇` 与字符串 `世界.同场角色` 在 Schema 根转换中迁移到新结构，转换结果不再保留旧字段，确保幂等。

## 多角色更新规则

### 角色进入

首次出现时先创建档案，再追加到出场数组：

```json
{ "op": "insert", "path": "/角色/雪之下雪乃", "value": { "作品": "我的青春恋爱物语果然有问题", "信任": 10, "好感度": 5 } }
{ "op": "insert", "path": "/世界/出场角色/-", "value": "雪之下雪乃" }
```

已有档案的角色再次进入时只追加姓名，不覆盖旧档案。Schema 去重作为防线，但更新规则仍要求模型先检查是否已在数组中。

### 角色离场

只从 `世界.出场角色` 删除对应数组项，不删除 `角色.[姓名]`。若同一回复有多人进出，可直接 `replace` 为完整的新数组，以避免连续索引变化导致误删。

### 同场互动

- 同一场景可以有任意数量角色同时在场。
- 每名角色的数值、心态、目标、判断和边界都从 `角色.[姓名]` 独立更新。
- 只更新在当前回复中实际感知到事件并产生反应的角色；不能因为同场而同步增加所有人的关系数值。
- 一次事件可影响多名角色，但每名角色必须有各自可观察依据。
- 未出场角色的档案保持不变，除非剧情明确提供其远程获知事件的依据。

## 状态栏数据流

1. 等待 `Mvu` 初始化。
2. 通过 `getAllVariables()` 和 `_.get(allVariables, 'stat_data.世界.出场角色', [])` 获取当前数组。
3. 当前选中角色只属于 UI 本地状态，不写入 MVU。
4. 根据选中姓名读取 `stat_data.角色.[姓名]`。
5. `VARIABLE_UPDATE_ENDED` 后重新读取数据；当前角色离场时自动选择数组第一项。
6. 最近角色提供快捷按钮，其余角色通过可搜索选择器访问，不产生页面级横向滚动。

## 视觉系统

封面与状态栏共享相同设计 token：

- 背景：中性冷黑，不使用大面积单一蓝紫或棕色。
- 强调：低饱和香槟金，仅用于当前选择、主要操作和关键数值。
- 材质：单一主磨砂容器，低对比细描边、轻微内高光和深层环境阴影。
- 圆角：主容器不超过 `8px`；小型控件使用 `6px`，避免夸张胶囊形状。
- 按钮：熟悉操作使用图标并提供 tooltip；无粗金边、外发光和高噪声渐变。
- 动效：悬停/按压 `180ms`，页面与面板进入 `320..480ms`，数值变化 `600ms`；支持 `prefers-reduced-motion`。

## 信息层级与文案

正式 UI 仅展示：场景、当前角色、当前关系指标、即时心态/目标、必要身份事实和明确操作。

以下内容不得出现在正式 UI：构建版本、CDN 地址、MVU/API 名称、初始化成功提示、变量路径、事务状态、开发说明、功能教程和重复标题。技术错误写入控制台；用户界面只显示可行动文案，例如“状态暂不可用”和重试按钮。

## 响应式布局

- 桌面封面：外壳 `16:9`，使用横向空间展示角色池和详情；状态栏采用紧凑双栏。
- 手机封面：外壳 `9:16`；状态栏单列，角色快捷入口最多四个，其他角色进入选择器。
- 平板：根据容器宽度在双栏与单栏之间切换，不依赖 viewport 字号缩放。
- 酒馆 iframe 内不使用 `vh` 强撑高度，不产生页面横向滚动或嵌套滚动。
- 固定格式控件使用稳定尺寸、网格轨道与最小宽度，动态文字采用截断或换行，禁止重叠。

## 测试与发布

- 回归测试覆盖：旧变量迁移、出场数组去重、多人同时在场、角色进出、每人独立变量、空场景和当前选择离场。
- 静态审查覆盖：无开发文案、无 `vh`、无横向页面滚动、MVU 初始化与更新监听完整。
- 浏览器审查覆盖桌面、平板和手机；检查封面流程、角色选择、多人状态栏切换、文本溢出、控制台错误和 reduced-motion。
- 发布时先提交并推送 UI commit，再把角色卡 CDN 固定到该 commit，重新生成角色卡后单独提交并推送，最后验证固定 CDN 返回 `200`。
