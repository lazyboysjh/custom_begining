## 数据标准

新增角色单独存放在 `plot/character_batches/recent_anime.yaml`，并在 `plot/character_sources.yaml` 注册 `expected_count: 50`。每人必须具备：

- 唯一 `id`、姓名及至少 3 个无冲突别名。
- 明确作品与采用的动画或原作时间点，不混合互相冲突的版本。
- 可识别外貌，而非只写抽象美貌词。
- 不少于 120 个汉字的角色介绍，说明身份、经历、性格矛盾、行动方式和口吻。
- 至少 3 条背景锚点、3 条关系锚点，第一条固定为与 `{{user}}` 开局无旧识。
- 独立的圣堂污秽钩子；钩子只解释跨界原因，不改写原作记忆、关系或能力。
- 原作或动画官方页面优先的来源记录；资料缺口才使用可靠二级资料，并在 notes 说明推导边界。
- 不新增统一年龄规则；没有可靠且必要的年龄信息时不输出空年龄字段。

## 界面文案

### 状态栏

- `枢纽抽卡` 改为 `召来新客`，表达新增当前场景角色，不使用游戏抽卡术语。
- `相聚一刻` 改为 `推进当前幕`，明确其会让所有在场角色共同响应当前场景。
- 执行时 `正在推进当前场景` 改为 `正在续写当前幕`。
- 空状态 `当前场景暂无角色。` 改为 `此刻尚无来客。`。
- 角色选择器 `全部角色` 改为 `切换来客`；上一位、下一位保留，但 aria-label 统一为上一位来客、下一位来客。
- 背景字段使用 `篇章 / 氛围 / 场所 / 能力`，避免“故事、圣堂”与字段值重复。
- 状态字段使用 `此刻 / 所求 / 观感 / 界限 / 异象`，让含义对普通用户可理解。

### 封面

- 保留 `圣堂初遇` 品牌名，删除带操控暗示或内部设定腔的冗余句。
- 首屏副文案改为“圣堂会回应一次尚未发生的相遇。设定你的能力，其余交给命运揭晓。”
- `进入圣堂` 改为 `开始初遇`。
- 能力页说明改为“选择你的净化能力。时代、场所、故事与来客将在下一步揭晓。”
- `一键随机开始` 改为 `随机启程`；`生成开局` 改为 `揭晓初遇`。
- 揭晓页 `圣堂启示` 改为 `来客将至`；生成标题改为 `正在编织初遇`。
- 取消按钮改为 `返回重选`，只描述用户动作，不暴露生成事务。
- 错误、等待与完成文案只说明当前状态和可执行动作，不展示 API、MVU、事务、变量、解析等内部词。

## 多角色与随机规则

- 开局仍随机选择一名首位来客。
- 状态栏的 `召来新客` 从 200 人池中排除已在场角色后随机选取，先建立角色档案，再向 `世界.出场角色` 数组追加姓名。
- `推进当前幕` 不新增角色，只生成一次包含所有在场角色的共同场景推进。
- 角色选择器完整支持 200 人数据，但默认只展示在场角色，搜索弹层才读取全部候选，避免状态栏拥挤。

## 验收标准

- 构建后角色数、唯一 id、唯一姓名均为 200。
- 新增批次恰好 50 人，来源记录与角色 id 一一对应。
- 200 份世界书角色档案存在，卡内角色绿灯为 200 条。
- 封面与状态栏不存在旧文案 `枢纽抽卡`、`相聚一刻`、`一键随机开始`、`生成开局`。
- 桌面 1280×720、平板 768×1024、手机 360×800 下，封面、揭晓页、状态栏无横向溢出、遮挡、裁切和越界控件。
- 浏览器实际点击 `揭晓初遇` 可进入揭晓页并完成 0 楼 MVU 提交；状态栏两个命令生成正确提示词。
- 全量回归、Superpower 整卡审查、发布审查均无错误。
- 静态资源提交先推送，再将卡内 CDN 固定到该提交完整 SHA 并提升缓存版本。
