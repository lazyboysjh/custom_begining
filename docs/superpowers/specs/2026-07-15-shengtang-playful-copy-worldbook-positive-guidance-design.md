# 圣堂萌系文案与世界书正向引导设计

## 目标

- 封面与状态栏按钮采用轻松、俏皮、偏萌系的措辞，保持动作含义一眼可懂。
- 世界书继续约束 OOC、媚主、元叙事和数值失控，但以可执行的正向行为指令为主。
- 保持 200 人目录、MVU 多角色结构、开局事务和 CDN 发布机制不变。

## 文案设计

封面主要操作使用短句：`开始召唤啦~`、`命运随便摇~`、`看看谁来啦`、`再想一下`。状态栏使用 `再摇一位~`、`故事继续走~`、`看看其他人`。加载状态围绕“来客登场”和“故事编织”表达，不出现 API、变量、初始化、事务等内部术语。

萌系仅用于按钮、加载状态和轻提示。错误提示保持直接、清晰，避免玩笑掩盖问题。波浪号只出现在短按钮文案中，控制密度，避免幼稚化。

## 世界书设计

世界书按写卡知识库的多角色卡规则审查：世界观与规则常驻，角色速览常驻，角色详情关键词触发；角色详情位于角色定义后，关键词覆盖正式名与别名，扫描深度为 2，并阻断递归。

提示词采用“目标行为 + 判断依据 + 可观察动作”的正向结构。例如：

- `禁止整段围着讨好转` 改为 `每轮先呈现角色自己的目标与风险判断，再回应{{user}}`。
- `不得改写性格底色` 改为 `数值只调节反应强度，口吻、职责与核心关系持续按档案表现`。
- `不奉承{{user}}` 改为 `角色只依据亲历且可验证的行为形成评价；认可具体事实，关系变化对应明确事件`。
- `禁止元叙事` 改为 `叙事始终停留在故事世界内，界面与变量信息仅供状态处理`。

角色模板中的 `不可违背` 改为 `演绎锚点`。全局规则只保留必要的事实边界，不以连续禁令堆叠。构建源文件是唯一事实源，生成后的 200 个角色文件与卡片随构建同步。

## 验证

- 自动测试断言萌系按钮文案存在、旧严肃文案消失。
- 自动测试统计世界书与角色模板中的禁令词，确保生成模板和核心规则完成正向改写。
- 运行 67 项以上整卡回归、Superpower 审查、构建和发布审查。
- 浏览器覆盖桌面、平板、手机的封面配置页、揭晓页和状态栏，检查溢出、裁切、点击与控制台错误。
